# Setup Guide

This guide explains how to install, run, verify, and reset the ReqSmell frontend from a
fresh machine or a fresh clone.

## What You Are Setting Up

The frontend is a browser application built with:

| Tool | Purpose |
|---|---|
| React | User interface framework |
| TypeScript | Type checking |
| Vite | Development server and production build |
| Redux Toolkit | Application state management |
| Tailwind CSS | Styling |
| Recharts | Charts for reports |
| jsPDF and html2canvas | Future client-side PDF report generation |
| Vitest | Unit and component test runner |

Version 1 is a verified baseline. It is ready for feature development, testing, version
switching, and GitHub work.

## Required Software

Install these before running the project:

| Software | Recommended version | How to check |
|---|---|---|
| Node.js | 20.19 or newer, or 22.13 or newer | `node --version` |
| npm | 10 or newer | `npm --version` |
| Git | Current stable version | `git --version` |
| PowerShell | Windows PowerShell or PowerShell 7 | `$PSVersionTable.PSVersion` |

The project has also been run on Node `22.11.0`, but newer Node versions avoid package
engine warnings from some modern tooling.

## Clone The Repository

Use this if you do not already have the project locally:

```powershell
git clone https://github.com/Mikun07/LLM_Prototype_Frontend.git
cd LLM_Prototype_Frontend
```

Use this if you are already inside the project folder:

```powershell
cd "C:\Users\ayomi\OneDrive\Documents\Thesis Work\LLM_Prototype\Frontend"
```

## Install Dependencies

For a clean install from `package-lock.json`, run:

```powershell
npm ci
```

Use `npm ci` when:

| Situation | Why |
|---|---|
| You cloned the project for the first time | It installs exactly what the lockfile defines |
| You switched versions | It restores dependencies for that version |
| You want a clean slate | It removes and recreates `node_modules` |
| You are preparing CI or GitHub Actions | It is more reproducible than `npm install` |

Use `npm install` when:

| Situation | Why |
|---|---|
| You changed `package.json` dependencies | It updates `package-lock.json` |
| You intentionally add, remove, or upgrade packages | It records dependency changes |

## Start The Development Server

Run:

```powershell
npm run dev
```

Expected result:

```text
Local: http://127.0.0.1:5173/
```

Open that URL in your browser.

Stop the server with:

```text
Ctrl+C
```

If port `5173` is already busy, Vite may choose another port. Use the URL printed in the
terminal.

## Run A Production Build

Run:

```powershell
npm run build
```

This does two things:

| Step | Meaning |
|---|---|
| `tsc --noEmit` | Checks TypeScript without generating extra root files |
| `vite build` | Creates the production output in `dist/` |

The `dist/` folder is generated output and is ignored by Git.

## Preview The Production Build

After building, run:

```powershell
npm run preview
```

Use this to inspect the built production version locally.

## Backend Setup Assumption

This repository is only the frontend. It does not start the backend.

The frontend expects the backend to run at:

```text
http://localhost:8000
```

The Vite proxy sends:

```text
/api/*
```

to:

```text
http://localhost:8000/api/*
```

If the backend is not running, the starter page still opens, but real API workflows will
fail once those features are implemented.

## First-Time Verification

After installing dependencies, run:

```powershell
npm run type-check
npm run lint
npm run test -- --run
npm audit
npm run build
```

Expected result:

| Command | Expected result |
|---|---|
| `npm run type-check` | No TypeScript errors |
| `npm run lint` | No ESLint errors |
| `npm run test -- --run` | All Vitest tests pass |
| `npm audit` | `found 0 vulnerabilities` |
| `npm run build` | Vite creates `dist/` successfully |

## Clean Local Setup

Use this when dependencies or generated files feel stale:

```powershell
git status
npm run version:use -- -Latest -CleanIgnored -Install
```

This removes ignored generated files and reinstalls dependencies.

It does not remove tracked source files.

## Files That Should Not Be Committed

These are generated and ignored:

| Path | Why ignored |
|---|---|
| `node_modules/` | Installed dependencies |
| `dist/` | Production build output |
| `dev-server.log` | Local development log |
| `*.tsbuildinfo` | TypeScript build cache |
| `vite.config.js` and `vite.config.d.ts` | Generated by older build behavior |
| `tailwind.config.js` and `tailwind.config.d.ts` | Generated by older build behavior |

Check before committing:

```powershell
git status --short --ignored
```
