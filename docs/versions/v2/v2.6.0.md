This is an interface design change because the run workflow now uses real backend status
responses and displays backend errors in the UI.

## Plain-Language Summary

Version `v2.6.0` connects the upload and analysis workflow to the FastAPI backend. The
frontend now uses backend CSV upload, creates real backend analysis runs, polls backend
status, and shows provider or pipeline errors when the backend reports them. This is the
first frontend version where the run screen is no longer a local simulation.

## What Changed From The Previous Version

| Area | What changed | Why |
|---|---|---|
| Upload | CSV upload now uses `POST /api/upload` | Backend status codes and parsing rules are now authoritative |
| Analysis | Run button now calls `POST /api/analyse` | Creates real backend analysis runs |
| Polling | Run screen polls `GET /api/status/{runId}` | Shows live backend pipeline progress |
| Error handling | API errors are converted to readable messages | Users can understand `503`, validation errors, provider quota errors, and pipeline failures |
| Dashboard | Dashboard supports partial reports | Single-model runs do not require a comparison report |
| Version | Package version updated to `2.6.0` | Documents the backend integration change |

## What Is Included

| Area | Details |
|---|---|
| Backend upload | `useFileUpload` calls the backend upload endpoint |
| Backend analysis | `useAnalysisRun` starts and polls backend runs |
| HTTP status handling | `503` provider errors and validation messages display in the run UI |
| Quota handling | OpenAI `429 insufficient_quota` is shown as a clear billing/quota message |
| Partial reports | Claude-only, ChatGPT-only, and comparison reports render when available |

## Backend Assumption

This version expects backend `v1.1.0` or later running at:

```text
http://localhost:8000
```

The Vite proxy forwards frontend `/api` calls to that backend.

## Known Limitations

| Limitation | Impact |
|---|---|
| Live OpenAI quota is still managed outside the app | The app explains quota errors, but billing still has to be fixed in OpenAI Platform |
| No backend run list endpoint yet | The frontend can poll a known run but cannot list all previous runs |
