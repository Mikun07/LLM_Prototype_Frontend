This version is an interface design change because API errors now appear in a shared
top-right toast alert system instead of only being shown inside the current step.

## Plain-Language Summary

Version `v2.7.0` adds a global toast alert system for frontend error handling. Upload
errors, backend analysis start errors, and backend analysis run failures now display as
top-right alerts for 7 seconds. This makes important failures easier to notice even when
the workflow changes steps or the user is focused on another part of the page.

## What Changed From v2.6.0

| Area | What changed | Why |
|---|---|---|
| Alerts | Added a shared `ToastViewport` rendered from `App` | Every step can show alerts in the same top-right location |
| State | Added a Redux `toast` slice with add, dismiss, and clear actions | Alerts can be triggered from hooks and future components |
| Upload | Failed CSV uploads now create an error toast | Upload parsing or backend validation errors should be immediately visible |
| Analysis | Failed run creation and failed backend runs now create error toasts | Provider errors such as `429 insufficient_quota` need a clear alert |
| Accessibility | Error toasts use `role="alert"` and non-error toasts use `role="status"` | Screen readers receive alert updates correctly |
| Version | Package metadata and version documentation updated to `2.7.0` | Keeps this visible UI change restorable as its own version |

## What Is Included

| Area | Details |
|---|---|
| `src/components/shared/ToastViewport.tsx` | Fixed top-right toast stack with close buttons and tone-specific icons |
| `src/store/slices/toastSlice.ts` | Redux queue for `error`, `warning`, `info`, and `success` alerts |
| `src/hooks/useFileUpload.ts` | Dispatches an upload error toast when backend upload fails |
| `src/hooks/useAnalysisRun.ts` | Dispatches analysis error toasts for start failures and run failures |
| Auto-dismiss | Toasts dismiss automatically after 7 seconds |
| Queue size | The alert queue keeps the most recent four toasts to avoid covering the page |

## Backend Assumption

This version expects backend `v1.1.0` or later running at:

```text
http://localhost:8000
```

The Vite proxy forwards frontend `/api` calls to that backend. Backend error responses are
converted by `src/api/client.ts` before they are displayed in the toast alert.

## Known Limitations

| Limitation | Impact |
|---|---|
| Toasts are not persisted | Refreshing the page clears visible alerts |
| Toasts are frontend-only | Backend logs still need to be checked for server-side debugging |
| `v2.7.0` tag must be created after commit | Version switching to `v2.7` works only after the tag exists |

## Upgrade Notes

Upgrading from `v2.6.0` to `v2.7.0`:

- Upload and analysis failures now appear as top-right alerts for 7 seconds.
- Existing inline error messages remain available in their original step.
- No dependency installation changes are needed beyond the normal `npm ci`.

## Downgrade Notes

Returning from `v2.7.0` to `v2.6.0`:

```powershell
npm run version:use -- -Version v2.6 -CleanIgnored -Install
```

- Global toast alerts are removed.
- Upload and analysis errors still appear inline through the v2.6 backend status handling.
- The package version returns to `2.6.0`.
