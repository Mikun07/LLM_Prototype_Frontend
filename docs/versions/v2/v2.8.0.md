## Tier Classification

| Tier | Pattern | This version |
|---|---|---|
| Major baseline | `vX.0.0` - system architecture change | No |
| Interface design change | `vX.Y.0` - visible design or flow change | **Yes** |
| Patch | `vX.Y.Z` - minimal, barely noticeable | No |

This version is an interface design change because the gradient top-strip borders are
removed from every card across all steps and the dashboard, and the run flow now navigates
to the dashboard only when analysis is genuinely complete.

## Plain-Language Summary

Version `v2.8.0` removes the coloured gradient top-border strips that appeared on every
card in the wizard steps and dashboard, giving the UI a cleaner appearance. The test-only
"Preview result" shortcuts that loaded mock data are removed. A real "View Results" button
now appears on the Run step only when all selected pipelines have completed successfully,
and navigates to the Dashboard with actual results. New utilities and test files are also
added.

## What Changed From v2.7.0

| Area | What changed | Why |
|---|---|---|
| Design | Removed `h-1 bg-gradient-to-r` top-strip div from all cards | Cleaner card appearance without coloured border |
| Dashboard cards | Strip removed from `ReportHeader`, `StatCard`, `DonutChart`, `BarChart`, `BreakdownTable`, `ResultsSection` | Consistent with design direction |
| Step cards | Strip removed from `UploadStep`, `ConfigureStep` (`PanelStrip` now returns `null`), `RunStep` (both header and pipeline cards) | Consistent with dashboard |
| Run flow | "Preview result" mock-data shortcut button removed from `UploadStep` | Was a dev-only testing shortcut, not part of the real flow |
| Run flow | "Preview result" mock-data shortcut button removed from `RunStep` | Same reason |
| Run flow | "View Results" button added to `RunStep`, shown only when `allDone === true` | Gives user a clear call-to-action after real analysis completes |
| Utilities | `src/utils/reportPdf.ts` added | PDF report generation logic |
| Utilities | `src/utils/sampleDashboardResult.ts` added | Sample data for dashboard development |
| Components | `src/components/shared/MetadataGuide.tsx` added | Reusable metadata guidance component |
| Assets | `src/assets/pdf-header.png` added | Header image for PDF reports |
| Tests | `__tests__/api/client.test.ts` added | Tests for API client error handling |
| Tests | `__tests__/utils/sampleDashboardResult.test.ts` added | Tests for sample dashboard data utility |
| Imports | Removed `BarChart3`, `completeRun`, `sampleDashboard*` imports from `UploadStep` and `RunStep` | Dead code after mock button removal |

## Upgrade Notes

Upgrading from `v2.7.0` to `v2.8.0`:

- Gradient top-border strips are gone from all cards.
- No mock "Preview result" shortcut button remains on the Upload or Run steps.
- After analysis completes, use the "View Results" button to navigate to the dashboard.
- No dependency changes — `npm ci` is sufficient.

## Downgrade Notes

Returning from `v2.8.0` to `v2.7.0`:

```powershell
npm run version:use -- -Version v2.7 -CleanIgnored -Install
```

- Gradient top-border strips reappear on all cards.
- Mock "Preview result" buttons return on Upload and Run steps.
- "View Results" button is removed from Run step.
