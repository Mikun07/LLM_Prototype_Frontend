This is an interface design change. The configure step is fully redesigned with a new
selection flow, the version documentation folder is restructured, and automated version
monitoring is introduced via a pre-commit Git hook.

## Plain-Language Summary

Version `v2.4.0` makes three significant improvements to the project.

First, the configure step is redesigned so the user can select which AI model to use
(Claude, ChatGPT, or both) and which smell types to detect (ambiguity, inconsistency,
or both) using clear toggle cards. Each card has a plain-English description. The
advanced settings (temperature and batch size) are explained with tips that any
non-technical reader can understand.

Second, the version documentation folder is restructured. Version documents are now
organised by major version in subfolders (`docs/versions/v1/`, `docs/versions/v2/`),
making it easy to find all releases for a given major version in one place.

Third, a pre-commit Git hook is introduced that automatically validates the version
number, tier, and documentation before every commit is allowed. This prevents
undocumented or incorrectly classified versions from being committed.

## What Changed From v2.3.0

| Area | What changed | Why |
|---|---|---|
| ConfigureStep UI | Replaced pipeline summary cards with LLM selection toggles and smell type toggles | Give the user control over which model and smell type to run |
| ConfigureStep — advanced settings | Temperature and batch size sliders now have plain-English labels and explanatory tips | Non-technical users need to understand what these settings do |
| ConfigureStep — review text | Review confirmation text now reflects the actual selected models and smell types | Confirmation should describe what will actually happen |
| `RunConfig` type | Added `selectedModels` and `selectedSmellTypes` fields | Support the new per-run selection |
| `wizardSlice` | Added default values for `selectedModels` and `selectedSmellTypes` | Redux state must initialise the new fields |
| `RequirementType` | Changed `'FR' \| 'NFR' \| 'UNKNOWN' \| string` to `'FR' \| 'NFR' \| 'UNKNOWN' \| (string & {})` | The old union collapsed the literal types, making them invisible to the type checker |
| Version docs folder | Moved all version docs into `docs/versions/v1/` and `docs/versions/v2/` subfolders | Group releases by major version for clarity |
| `docs/VERSIONING.md` | Rewritten to document the three-tier model (major / design change / patch) | Previous guide described only two levels |
| `docs/versions/TEMPLATE.md` | Updated to include Tier classification table and "What changed" section | Template must match the new documentation standard |
| `docs/versions/index.md` | Updated with new subfolder links and tier column | Index must reflect the new folder structure |
| `docs/SETUP.md` | Added hook installation step | Every developer must install the hook after cloning |
| `README.md` | Updated version links and version model summary | Links pointed to old flat file paths |
| `scripts/check-version.ps1` | New script — validates version number, tier, doc existence, doc completeness, index entry | Enforce the versioning standard automatically |
| `scripts/install-hooks.ps1` | New script — installs the pre-commit hook into `.git/hooks/` | Hook installation must be repeatable and documented |
| `package.json` | Added `version:check` and `hooks:install` npm scripts | Make both tools runnable without knowing the script path |

## Backend Assumption

The frontend expects the backend at `http://localhost:8000`. Vite proxies all `/api/*`
requests during development. This version uses simulated progress and deterministic
preview reports — no live backend calls are required to use the interface.

## Known Limitations

| Limitation | Impact |
|---|---|
| Live LLM calls not wired | Dashboard shows deterministic preview data, not real model output |
| `selectedModels` and `selectedSmellTypes` not sent to backend | Backend API does not yet use the selection — reserved for `v3.0.0` |
| PDF reports not restyled | PDF export uses previous design tokens |
| Pre-commit hook not auto-installed | Every developer must run `npm run hooks:install` once after cloning |

## Upgrade Notes

Upgrading from `v2.3.0` to `v2.4.0`:

- Run `npm run hooks:install` once to activate the pre-commit version check
- `RunConfig` now has two new required fields: `selectedModels` and `selectedSmellTypes` — any code reading `config` must account for these
- Version docs have moved from `docs/versions/vX.Y.Z.md` to `docs/versions/vX/vX.Y.Z.md` — update any bookmarks or links

## Downgrade Notes

Returning from `v2.4.0` to `v2.3.0`:

```powershell
npm run version:use -- -Version v2.3 -CleanIgnored -Install
```

- The `check-version.ps1` and `install-hooks.ps1` scripts will not exist on `v2.3.0`
- The pre-commit hook will still be present in `.git/hooks/` but will fail to find the script — remove it manually if needed: `Remove-Item .git\hooks\pre-commit`
- `RunConfig` reverts to two fields only (`temperature`, `maxGroupSize`)
- Version docs revert to the flat `docs/versions/` structure
