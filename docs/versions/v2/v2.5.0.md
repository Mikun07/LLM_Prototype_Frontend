This version is an interface design change because the upload step gains a visible
pagination workflow. Users can now inspect the full uploaded CSV before continuing,
instead of being limited to the first preview rows.

## Plain-Language Summary

Version `v2.5.0` fixes the CSV upload preview so users can page through uploaded
requirements immediately after selecting a file. It adds rows-per-page controls,
previous and next buttons, and a clear row range label. It also adds a lint rule that
blocks direct `window` usage so browser-global code consistently uses `globalThis`.

## What Changed From v2.4.0

| Area | What changed | Why |
|---|---|---|
| Upload preview | Replaced the fixed first-20-row preview with a paginated table | Users need to inspect every row in the uploaded CSV |
| Upload preview controls | Added rows-per-page, previous, next, and page count controls | Navigation must be available before the configure step |
| Pagination hook | Clamps the current page when the row set or page size changes | Replacing a CSV should not leave the preview on an empty stale page |
| Detected columns | Removed Yes/No prefixes and use green/red chip color to show present/missing columns | Keep the upload status compact and easier to scan |
| ESLint policy | Added a `no-restricted-globals` guard for `window` | Prevent future "Prefer globalThis over window" issues |
| Visible version | Updated package metadata, lockfile, README, version index, and app version label to `2.5.0` | Keep the working version documented and switchable |

## Backend Assumption

The frontend still uses deterministic local preview reports. The upload preview
pagination is fully client-side and does not require the backend.

## Known Limitations

| Limitation | Impact |
|---|---|
| Pagination is client-side | Very large CSV files are still loaded into browser memory |
| Upload pagination does not search/filter rows | Search remains available in dashboard result tables, not the upload preview |
| `v2.5.0` tag must be created after commit | Version switching to `v2.5` works only after the tag exists |

## Upgrade Notes

Upgrading from `v2.4.0` to `v2.5.0`:

- Upload a CSV with more than 10 rows to see the new pagination controls.
- Run `npm run lint`; any direct `window` usage now fails and should be replaced with `globalThis`.
- Run `npm run version:check` before committing so the version document and index stay aligned.

## Downgrade Notes

Returning from `v2.5.0` to `v2.4.0`:

```powershell
npm run version:use -- -Version v2.4 -CleanIgnored -Install
```

- The upload preview reverts to the old fixed preview behavior.
- The `window` lint guard is not present in `v2.4.0`.
- The visible app version label returns to `v2.4.0`.
