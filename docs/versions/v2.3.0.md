## Plain-Language Summary

Version `v2.3.0` is a visual and code-quality update built on the `v2.0.0` interface
baseline. It replaces the flat colour scheme with a richer multi-colour design system,
removes the fixed minimum viewport width so the app works on smaller screens, adds a
custom SVG logo to the header, and resolves every ESLint and SonarLint warning across
all source files.

The interface structure, routing, Redux state shape, API contract, and backend assumptions
are unchanged from `v2.2.0`.

## Version Classification

| Classification | Use when |

## Backend Assumption

The frontend expects the backend at `http://localhost:8000`. Vite proxies all `/api/*`
requests during development. This version uses simulated progress and deterministic
preview reports — no live backend calls are required to use the interface.

## Known Limitations

| Limitation | Impact |
|---|---|
| No live LLM calls | Dashboard shows deterministic preview data, not real model output |
| PDF reports not restyled | PDF export uses the previous design tokens |
| No favicon | Browser tab shows default icon; logo exists only in the app header |

## Upgrade Notes

Upgrading from `v2.2.0` to `v2.3.0`:

- No state shape changes — Redux store structure is identical
- No API contract changes — backend endpoints and request/response types unchanged
- `tsconfig.json` target is now ES2021; all modern browsers support this
- `tailwind.config.ts` has a larger colour palette; any custom CSS using old colour
  names (e.g. `brand-500`) is unaffected as those values still exist

## Downgrade Notes

Returning from `v2.3.0` to `v2.2.0`:

```powershell
npm run version:use -- -Version v2.2 -CleanIgnored -Install
```

- The `ReqSmellLogo` component will not exist on `v2.2.0`
- The extended Tailwind palette will revert to the original flat blue scheme
- The `min-width: 1024px` body lock will be restored
- tsconfig target reverts to ES2020
