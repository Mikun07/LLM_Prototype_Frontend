## Plain-Language Summary

Version `v2.1.0` does not replace the version-2 interface baseline. It clarifies the
version-control policy for all future work:

| Rule | Meaning |
|---|---|
| `v1.0.0` | Frozen environment baseline |
| `v2.0.0` | Frozen interface baseline |
| `v2.1.0`, `v2.2.0`, `v2.3.0` | Normal changes and adjustments built from the version-2 baseline |
| `v3.0.0`, `v4.0.0` | New major baselines created only when project scope changes significantly |

## What Changed

| Area | Details |
|---|---|
| Version policy | Documents now state that ordinary changes after `v2.0.0` become `v2.1.0`, `v2.2.0`, and so on |
| Current version | `package.json`, lockfile, and visible app version move to `2.1.0` |
| Baseline handling | `v2.0.0` remains frozen and restorable |
| Future release guidance | Examples now point to `v2.2.0` as the next normal release after `v2.1.0` |
