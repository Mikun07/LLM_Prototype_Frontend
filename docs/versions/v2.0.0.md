## Plain-Language Summary

Version 2 turns the version-1 environment setup into a usable ReqSmell frontend
interface. A user can upload a CSV, review detected columns and preview rows, confirm the
fixed four-pipeline run configuration, watch progress, and inspect model and comparison
reports in a dashboard.

This version uses deterministic frontend preview reports so the interface can be reviewed
without a live backend result payload. The browser still does not call any LLM API and
does not persist requirement data to localStorage, sessionStorage, or cookies.

## What Is Included

| Area | Details |
|---|---|
| Upload interface | Drag-and-drop and file picker CSV upload |
| CSV parsing | Client-side parsing through PapaParse with validation and row normalization |
| CSV preview | File metadata, detected columns, and up to 20 preview rows |
| Configuration | Fixed Claude and ChatGPT pipelines for ambiguity and inconsistency |
| Settings | Temperature and max inconsistency group size controls |
| Run progress | Overall and per-pipeline progress cards |
| Dashboard | Claude, ChatGPT, and comparison tabs |
| Charts | Custom Recharts donut and bar chart wrappers |
| Tables | Search, filter, sort, pagination, badges, and expandable suggestions |
| Export | CSV export for result tables |
| Example data | `examples/sample-requirements.csv` for immediate interface testing |
| State | Redux Toolkit state for wizard, config, requirements, progress, and reports |
| Tests | CSV parser, report factory, and formatter tests |
| Versioning | Version docs updated for `2.0.0` with rollback to `v1.0.0` |

## Backend Assumption

The frontend API boundary still expects the backend at:

```text
http://localhost:8000
```

Version 2 does not depend on the backend to demonstrate the visible interface. Later
versions should replace preview report generation with live backend run results.

## Known Limitations

| Limitation | Impact |
|---|---|
| Preview report generation is frontend-only | Results are deterministic interface data, not LLM output |
| PDF export is not wired yet | CSV export works, but PDF report generation remains future work |
| Live backend status polling is not wired into the UI run | The API client exists, but the run step uses interface progress |
| Component test coverage is still limited | Utility tests pass, but UI interaction tests should be expanded |

## Upgrade Notes

Version 2 changes the product surface from a starter screen to a full wizard interface.
Run `npm ci` after switching from version 1 because `package.json` and `package-lock.json`
now declare version `2.0.0`.

## Downgrade Notes

Use `v1.0.0` if you need the original environment-only baseline. Downgrading removes the
version-2 interface files from the checked-out snapshot.
