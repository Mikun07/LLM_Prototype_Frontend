## Plain-Language Summary

Version 1 is the first clean frontend setup for ReqSmell.

It is not the complete user-facing ReqSmell application yet. It is the foundation that
lets a developer install the project, run the local app, run tests, check code quality,
build production output, switch versions, and restore a clean slate.

## Who This Version Is For

| Person | What they can do with version 1 |
|---|---|
| New developer | Install dependencies and run the starter frontend |
| Researcher | Confirm the frontend tooling baseline is reproducible |
| Maintainer | Create future versions from a clean starting point |
| Reviewer | Run type-check, lint, tests, audit, and build |

## First-Time Setup

Clone from GitHub:

```powershell
git clone https://github.com/Mikun07/LLM_Prototype_Frontend.git
cd LLM_Prototype_Frontend
```

Install dependencies:

```powershell
npm ci
```

Run the development server:

```powershell
npm run dev
```

Open:

```text
http://127.0.0.1:5173/
```

Stop the server with `Ctrl+C`.

## Restore Version 1

View the exact `v1.0.0` snapshot:

```powershell
git fetch --tags
git switch --detach v1.0.0
npm ci
```

Restore version 1 and remove generated files:

```powershell
npm run version:use -- -Version v1.0.0 -CleanIgnored -Install
```

Create an editable branch from version 1:

```powershell
npm run version:use -- -Version v1.0.0 -Branch work/from-v1.0.0 -Install
```

Return to latest `main`:

```powershell
npm run version:use -- -Latest -Install
```

## Dependency Notes

The planning document in `.AI_tool/frontend_README.md` described Vite 5 and jsPDF 2.x.
Version 1 uses patched packages where needed to keep `npm audit` clean.

| Package | Version line | Reason |
|---|---|---|
| `vite` | `6.4.2` | Avoids known dev-server advisory in older Vite/esbuild versions |
| `vitest` | `3.2.4` | Compatible with Vite 6 |
| `jspdf` | `4.2.1` | Avoids vulnerable transitive DOMPurify versions |
| `jspdf-autotable` | `5.0.7` | Compatible with jsPDF 4 |

## Backend Assumption

The frontend expects the backend at:

```text
http://localhost:8000
```

Vite proxies:

```text
/api/*
```

to:

```text
http://localhost:8000/api/*
```

The starter version can run without the backend, but future upload and analysis workflows
will need it.
